Описание

Командная оболочка (CLI) для взаимодействия с различными системными компонентами, такими как файловые системы, Wi-Fi, SD-карты и другие системные настройки. Система поддерживает множество команд для управления файлами, директориями и системными конфигурациями.

Система разработана для всего семейства микроконтроллеров ESP32, но специально адаптирована для модели ESP32-C3.

Установка

🔧

Стандартная установка через Arduino IDE

  1. Предварительные требования: Arduino IDE 2.x, ESP32 core 3.2.0+
  2. Клонируйте репозиторий:
    git clone https://github.com/VLPLAY-Games/XenoOS.git
  3. Откройте XenoOS.ino в Arduino IDE
  4. Выберите плату ESP32-C3 Dev Module
  5. Настройте параметры в Tools:
    • USB CDC On Boot: "Enabled"
    • CPU Frequency: "160MHz (WiFi)" (default value)
    • Core Debug Level: "None" (default value)
    • Erase All Flash Before Sketch Upload: "Disabled" (default value)
    • Flash Frequency: "80MHz" (default value)
    • Flash Mode: "QIO" (default value)
    • Flash Size: "4MB (32Mb)" (default value)
    • JTAG Adapter: "Disabled" (default value)
    • Partition Scheme: "Minimal SPIFFS (1.9MB APP with OTA/190KB SPIFFS)"
    • Upload Speed: "921600" (default value)
    • Zigbee Mode: "Disabled" (default value)
  6. Нажмите Upload для прошивки
⚡

Альтернативная установка для ESP32-C3

  1. Скачайте последний релиз с GitHub
  2. Для прошивки объединенного файла:
    esptool.py --chip esp32c3 --baud 921600 write_flash 0x0 XenoOS.merged.bin
  3. Или прошейте файлы отдельно:
    esptool.py --chip esp32c3 --baud 921600 write_flash \
      0x1000 XenoOS.bootloader.bin \
      0x8000 XenoOS.partitions.bin \
      0x10000 XenoOS.bin

Как использовать

1

Подключитесь к устройству через Serial порт (скорость 115200) в Putty или другом терминале

2

Введите команды напрямую в терминал

3

Используйте help для просмотра всех доступных команд

4

Для диагностики системы используйте:

  • system diagnostic - запуск диагностики
  • system check - проверка системы

Устранение неполадок

Если система не запускается или работает некорректно:

  1. Убедитесь, что используется правильная версия Arduino Core (3.2.0+)
  2. Проверьте подключение Serial порта (скорость 115200)
  3. Для SD карт используйте карты объемом не более 4GB
  4. Запустите system diagnostic для проверки компонентов
